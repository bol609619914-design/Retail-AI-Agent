# Changelog

本文件记录 Retail-AI-Agent 的重要更新。

格式参考 Keep a Changelog 的思路，并结合当前项目原型阶段做简化整理。

## [2026-04-07]

### 第二轮升级

- 将产品库扩展为更接近真实零售展示的数据模型，补充品牌、类别、价格区间、材质、工艺、专业参数与产品图片
- 将推荐结果升级为结构化顾问判断，新增“为什么选这款”“为什么暂不推荐别的”“命中的偏好点”“顾问判断摘要”
- 将后端对话节奏调整为三段式顾问流程：先确认空间，再收拢氛围或功能，最后给出单品推荐
- 将首页推荐区升级为更完整的作品级展示卡片，强化层次、留白、摘要区与品牌化信息呈现
- 在前端加入“本地演示模式 / 在线模型模式”状态提示，让无 Key 演示链路更加明确
- 升级 README，补充设计目标、系统架构图、推荐流程图，以及“为什么当前阶段使用 JSON”的设计说明

### 新增

- 新增中文主导的聊天首页交互，界面升级为“对话区 + 推荐信息区”的双栏布局
- 新增结构化推荐卡片展示，可同步呈现产品名称、核心功能、用户利益与适用场景
- 新增 `/chat` 的 SSE 流式响应格式，支持 `chunk`、`product`、`done` 等事件
- 新增无 OpenAI Key 时的本地中文 mock 顾问回复逻辑，方便演示与联调
- 新增针对中文输入的轻量检索 token 拆分逻辑，提升产品匹配稳定性

### 优化

- 将前端聊天体验调整为以中文文案为主，更贴合当前项目的目标用户与展示语境
- 优化推荐流程：当后端已具备足够上下文时，同步返回产品元数据，前端即时展示推荐卡片
- 优化本地运行链路，补齐 Python 3.11、后端虚拟环境与依赖安装后的可运行状态

### 修复

- 修复后端读取产品库时的路径错误，确保正确指向 `backend/data/products.json`
- 修复 `products.json` 含 UTF-8 BOM 时的读取异常，改为兼容 `utf-8-sig`
- 修复由于中文匹配策略过于保守导致推荐卡片未触发的问题
- 修复前后端在无 OpenAI Key 场景下无法完成完整演示的问题

### 说明

- 当前前端运行地址：`http://127.0.0.1:3000`
- 当前后端运行地址：`http://127.0.0.1:8000`
- 当前版本即使不配置 `OPENAI_API_KEY`，也可以完成中文流式推荐演示
